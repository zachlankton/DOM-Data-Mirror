    • Introduction

    zData is an experiment created in an attempt to accomplish a few tasks...

    • Decrease Code Required to create Basic Web Apps
    • Use as little abstraction as possible
    • Extend HTML, instead of generating it via templates and controllers
    • Reduce or eliminate controller code
    • Reduce or eliminate the need for frameworks
    • Reduce overhead

    zData, by itself, doesn't necessarily accomplish all of these things. However, zData is extendable and included in the code base are some extensions that start pushing zData in that direction.

    By writing some simple extensions you can considerably reduce the amount of boilerplate required in your app. We will show how this can be done in the examples to follow.
